# Paper 1: Command Area Irrigation Growth

## Research question

Over the past 20 years, has irrigation expansion in arid Sub-Saharan Africa occurred predominantly within areas supported by large-scale dam command areas, or has it shifted toward areas outside modeled large-dam infrastructure?

## Role of this folder

This is the canonical workflow for the command-area growth paper. Analyses elsewhere in the repository can support interpretation, but final paper claims should trace back to the notebooks, tables, and figures produced here.

## Canonical workflow

The canonical workflow is now script-first. Notebooks can be used for
presentation, exploration, and figure polish, but paper claims should trace back
to the CSVs generated by these scripts.

Run from the repository root:

```powershell
python Code/paper1_command_area_growth/00_prepare_paper_inputs.py
python Code/paper1_command_area_growth/01_prepare_yearly_command_areas.py --overwrite
python Code/paper1_command_area_growth/02_extract_irrigation_inside_outside.py --all-touched
python Code/paper1_command_area_growth/03_growth_decomposition.py
```

Or run the wrapper:

```bash
bash Code/paper1_command_area_growth/run_paper1_pipeline.sh
```

See `RUNBOOK.md` for the research estimand, assumptions, current blockers, and
expected outputs.

The notebook-facing paper workflow is:

1. `notebooks/00_data_inventory.ipynb`
   - Check that all raw and processed inputs exist.
   - Record years available for AEI, CPIS, dams, command areas, DEM, and aridity masks.
   - Write a machine-readable inventory table.

2. `00_prepare_paper_inputs.py`
   - Validate command-area and AEI raw inputs.
   - Build the SSA arid country mask if it is missing.
   - Build processed GDW arid-SSA and irrigation-dam layers if they are missing.

3. `01_prepare_yearly_command_areas.py`
   - Filter dams by commissioning year.
   - Keep only command areas attached to dams active by each analysis year.
   - Dissolve overlaps so pixels are counted once.
   - Save one merged command-area layer per year.

4. `02_extract_irrigation_inside_outside.py`
   - Mask gridded irrigation rasters by yearly command areas.
   - Compute irrigation inside and outside command areas by year and country.
   - Save the core inside/outside panel table.

5. `03_growth_decomposition.py`
   - Decompose net irrigation growth into inside-command-area and outside-command-area components.
   - Estimate what share of expansion occurred outside modeled large-dam service zones.

6. `notebooks/04_country_robustness.ipynb`
   - Repeat growth decomposition by country.
   - Test sensitivity to influential countries and sparse command-area coverage.

7. `notebooks/05_cpis_supporting_evidence.ipynb`
   - Compare CPIS inside/outside command areas using year-matched CPIS inputs.
   - Use CPIS as supporting evidence for decentralized irrigation, not as the primary outcome.

8. `notebooks/06_paper_figures_tables.ipynb`
   - Generate only final manuscript-ready figures and tables from saved intermediate CSVs.

## Legacy reference notebooks

The old exploratory notebooks have been preserved in `legacy_notebooks/`. They are useful references, but they should not be cited as final paper workflow unless their logic has been cleaned and rerun through the canonical notebooks above.

Especially relevant legacy notebooks:

- `7_Prep_Descriptive_Stats.ipynb`: yearly command-area preparation.

- `6_AEI_Ratio.ipynb`: country-year command-area irrigation summaries.
- `3_CA_Targeting_Ratio copy.ipynb`: CPIS targeting ratio prototype.

## Supporting evidence

The water-source notebooks in `Code/2_water_source_analysis/` support interpretation of outside-command-area irrigation, especially CPIS activity, dam accessibility, groundwater plausibility, and anomalies. They should be framed as supporting evidence rather than the main answer to this paper's research question.
